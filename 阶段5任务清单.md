# 阶段 5 任务清单

## 说明

本清单从 [实施说明.md](/C:/Codefield/DraftCore/实施说明.md) 的阶段 5 单独整理而来，用于指导阶段 6 的实际开发与试运行。

本清单沿用既有约束：

- 仅围绕个人、本地、报告优先的 MVP 展开
- 任务按闭环小步推进，每次只解决一个关键问题
- 优先验证素材归集与复用，再进入起草、改写和归档
- 每个任务完成后都应能用真实样例演示一次

## 总体推进原则

### 任务拆分原则

1. 每个任务只覆盖一段明确的使用路径，不把多个目标混在同一轮实现中。
2. 每个任务都必须为下一个任务提供可复用输入或明确前置条件。
3. 每个任务默认只实现 MVP 所需的最小能力，不提前扩展高级功能。

### 统一完成定义

每个任务完成时，应同时满足以下条件：

1. 已有真实输入样例。
2. 已能展示处理过程或关键操作结果。
3. 已产生明确输出结果。
4. 输出结果可被你判断是否满足当前任务目标。

### 不纳入本清单的内容

以下内容不在阶段 5 任务清单范围内：

- PPT、Excel 独立流程
- 多人协作、审批、共享、评审
- 云端同步
- 复杂标签体系和复杂版本体系
- 高级自动生成、深度智能推荐、复杂统计分析

## 任务 1：报告项目与素材收集

### 任务目标

建立一条最基础的进入路径，使一个真实报告任务可以被创建为报告项目，并把相关原始素材、模板素材、参考成果纳入统一管理范围。

### 要解决的问题

- 当前报告任务缺少统一入口
- 素材分散在本地不同目录中，无法围绕任务集中管理
- 模板和历史成果虽然存在，但没有和当前任务建立明确联系

### 范围

纳入：

- 报告项目的建立
- 原始素材、模板素材、参考成果的纳入
- 当前任务与素材的基础关联

不纳入：

- 素材用途标记细化
- 复用内容提取
- 草稿生成和归档

### 建议输入

- 一个真实报告主题
- 一组相关文档、表格、图片或参考资料
- 至少一个模板文件或历史报告样例

### 执行动作

1. 创建一个报告项目，记录项目名称、主题和目标输出。
2. 选取当前报告相关的原始素材文件。
3. 选取可用模板素材和参考成果。
4. 建立报告项目与这些素材对象之间的基础关联。
5. 输出一个可查看的当前任务素材范围。

### 交付产出

- 一个可识别的报告项目
- 一组已纳入范围的素材对象清单
- 报告项目与素材对象的基础关联结果

### 前置条件

- 已明确一个真实报告任务
- 已准备至少一组真实本地素材

### 完成标志

- 能围绕一个真实任务建立报告项目
- 能把原始素材、模板素材、参考成果纳入同一个任务范围
- 能清楚展示“当前项目拥有哪些输入素材”

### 与后续任务的关系

本任务产出将作为任务 2 的输入基础。

## 任务 2：素材归集与用途标记

### 任务目标

把任务 1 中已纳入的分散素材组织成可使用的素材集合，并为素材补充最小必要用途说明，支持后续筛选与复用。

### 要解决的问题

- 素材已被收进来，但仍然分散、难以理解用途
- 当前任务中哪些素材适合做背景、数据、模板或参考尚不明确
- 后续主草稿生成缺少清晰输入上下文

### 范围

纳入：

- 素材集合建立
- 最小用途说明补充
- 已收集素材的组织化展示

不纳入：

- 深度标签体系
- 自动分类规则
- 复杂筛选条件设计

### 建议输入

- 任务 1 的报告项目
- 已纳入管理范围的原始素材、模板素材、参考成果

### 执行动作

1. 围绕当前报告主题建立素材集合。
2. 将相关素材归入该素材集合。
3. 为各素材补充最小必要用途说明，例如背景参考、数据来源、结构模板、历史参考。
4. 初步区分“已收集素材”和“当前更可能被使用的素材”。
5. 输出一个可查看的素材集合视图。

### 交付产出

- 一个围绕当前任务组织好的素材集合
- 一组带有基础用途说明的素材项
- 初步可用的素材输入上下文

### 前置条件

- 任务 1 已能展示报告项目及其素材范围

### 完成标志

- 能将分散素材归集为一个清晰的任务素材集合
- 能看出每个素材的大致用途
- 能为任务 3 提供更明确的复用定位输入

### 与后续任务的关系

本任务产出将作为任务 3 和任务 4 的输入基础。

## 任务 3：复用定位

### 任务目标

验证系统能从模板素材和参考成果中定位当前报告可复用的结构或内容，并将这些复用结果关联到当前报告项目。

### 要解决的问题

- 历史报告和模板存在，但复用入口不清晰
- 无法快速找出可直接借用的结构、章节或内容片段
- 起草前缺少现成骨架和复用来源

### 范围

纳入：

- 从模板和历史成果中查找可复用内容
- 记录复用结果与当前报告项目的关联
- 支持为主草稿生成准备复用输入

不纳入：

- 大规模自动摘要
- 深度语义推荐
- 自动生成完整正文

### 建议输入

- 任务 2 产出的素材集合
- 至少一个模板素材
- 至少一个参考成果

### 执行动作

1. 在模板素材中定位可复用的结构骨架、章节模板或表达方式。
2. 在参考成果中定位可复用的历史内容片段或结构参考。
3. 将这些复用结果关联到当前报告项目。
4. 输出“当前项目可复用内容清单”。

### 交付产出

- 一组可复用结构或内容定位结果
- 复用结果与当前报告项目的关联信息
- 为主草稿生成提供的复用输入

### 前置条件

- 任务 2 已形成可查看的素材集合与用途说明

### 完成标志

- 能明确展示至少一组模板复用结果
- 能明确展示至少一组历史成果复用结果
- 能让你判断“这些内容是否足够支撑起草”

### 与后续任务的关系

本任务产出将直接进入任务 4。

## 任务 4：主草稿生成

### 任务目标

基于已确认素材和复用结果，生成一个可继续演进的报告主草稿骨架或初稿，验证“边收边写”的起草入口。

### 要解决的问题

- 当前写作常常从零开始，缺少稳定起草入口
- 素材和复用内容已经具备，但尚未形成实际草稿
- 报告项目尚未进入可编辑、可迭代状态

### 范围

纳入：

- 主草稿建立
- 草稿骨架生成
- 使用已确认素材和复用结果生成初稿

不纳入：

- 全自动长篇写作
- 多草稿并行体系
- 复杂版本控制

### 建议输入

- 任务 2 的素材集合
- 任务 3 的复用结果

### 执行动作

1. 建立当前报告项目的主草稿。
2. 结合模板结构、历史参考和已确认素材，生成主草稿骨架。
3. 在骨架基础上补入最小初稿内容。
4. 输出一个可继续改写的主草稿版本。

### 交付产出

- 一个属于当前报告项目的主草稿
- 一份可查看的草稿结构或初稿内容

### 前置条件

- 任务 3 已完成可复用内容定位

### 完成标志

- 能看到主草稿的结构或初稿正文
- 主草稿内容已明确来源于素材和复用结果，而不是完全手工脱离上下文生成
- 主草稿已可作为任务 5 的改写对象

### 与后续任务的关系

本任务产出将直接进入任务 5。

## 任务 5：主草稿改写

### 任务目标

在一个主草稿上完成补充、改写、合并和统一表达，验证主草稿持续演进而非重建的工作方式。

### 要解决的问题

- 报告编写过程中需要频繁改写和统一表达
- 草稿内容需要补充新素材并做结构或语气调整
- 现有流程缺少一条稳定的“写中优化”路径

### 范围

纳入：

- 草稿补充
- 草稿改写
- 内容合并与统一表达

不纳入：

- 多版本对比管理
- 高级润色策略
- 复杂协同编辑

### 建议输入

- 任务 4 生成的主草稿
- 当前任务仍在使用的素材集合

### 执行动作

1. 在主草稿中补充缺失内容。
2. 对不清晰或不统一的表达进行改写。
3. 对重复或分散内容进行合并整理。
4. 形成一个可验收的主草稿版本。

### 交付产出

- 一份经过改写和整合的主草稿
- 一次可演示的草稿优化结果

### 前置条件

- 任务 4 已能输出主草稿

### 完成标志

- 能清楚展示改写前后差异
- 能证明主草稿是在原有基础上演进，而不是重新从零创建
- 主草稿已满足进入最终归档的条件

### 与后续任务的关系

本任务产出将直接进入任务 6。

## 任务 6：最终报告归档

### 任务目标

输出最终报告，并保留其与素材、模板、参考成果之间的基础关联关系，验证第一版 MVP 已具备最小复用闭环。

### 要解决的问题

- 报告完成后缺少稳定沉淀方式
- 结果与输入素材脱节，后续难以复用
- 阶段 4 要求的“基础归档能力”尚未真正闭环

### 范围

纳入：

- 最终报告输出
- 最终报告归档
- 与素材、模板、参考成果的基础关联保留

不纳入：

- 重型归档卡片
- 统计分析
- 自动知识沉淀体系

### 建议输入

- 任务 5 的可验收主草稿
- 当前项目使用过的素材、模板、参考成果记录

### 执行动作

1. 将主草稿转为最终报告。
2. 对最终报告执行基础归档。
3. 保留最终报告与所用素材、模板、参考成果的基础关联。
4. 输出一个可回溯的归档结果。

### 交付产出

- 一份最终报告
- 一条基础归档记录
- 一组与素材和复用来源的关联关系

### 前置条件

- 任务 5 已形成可验收主草稿

### 完成标志

- 能展示最终报告已完成归档
- 能查看最终报告对应使用了哪些素材、模板和参考成果
- 能证明下一次新报告可以把该成果再次作为参考成果使用

## 建议执行顺序

建议严格按以下顺序执行：

1. 任务 1：报告项目与素材收集
2. 任务 2：素材归集与用途标记
3. 任务 3：复用定位
4. 任务 4：主草稿生成
5. 任务 5：主草稿改写
6. 任务 6：最终报告归档

## 阶段 5 验收

当以下条件全部满足时，可认为阶段 5 的任务清单已经足够支撑阶段 6：

1. 六个任务都有清晰目标、输入、动作、产出和完成标志。
2. 任一任务都能被单独演示和验收。
3. 相邻任务之间有明确承接关系。
4. 全部任务串联后，能够覆盖“素材归集 -> 复用定位 -> 主草稿生成 -> 主草稿改写 -> 最终归档”的最小闭环。
